## はじめに

これは、九州産業大学理工学部情報科学科下川研究室で利用している週間報告のテンプレートリポジトリである。
下川研では、週間報告を作成する標準的な環境として、LaTeX と GitHub を利用する。
文章作成に LaTeX、バージョン管理およびバックアップとして GitHub を使う。

## 🚀 クイックスタート

### 前提条件
- Docker Desktop がインストールされていること
  - Windows: WSL + Docker Desktop
  - macOS: Docker Desktop

### ワンライナーセットアップ

以下のコマンドを実行するだけで、週報リポジトリの作成から LaTeX 環境構築まで自動で行われる。

```bash
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/smkwlab/thesis-management-tools/main/create-repo/setup-wr.sh)"
```

このコマンドで以下が自動的に実行される。
1. GitHub 認証（ブラウザで認証）
2. 自分専用の週報リポジトリ作成（`k99rs999-wr` 形式）
3. LaTeX 環境の自動構築（devcontainer）
4. Git の初期設定

セットアップ完了後は、作成されたディレクトリで VS Code を開いて週報の作成を開始できる。

## 1. 基本的な使い方

週間報告を執筆する上では、毎週 tex ファイルを作成する。

### 推奨手順
1. [クイックスタート](#🚀-クイックスタート推奨)でリポジトリを作成
2. VS Code で開いて週報を執筆
3. 完成したら GitHub に push

## 2. 具体的な作業内容

以下の説明の section の枝番号は 1. の箇条書きの番号と対応している。

### 2.1 自分用のリモートリポジトリの作成

#### 自動セットアップ
上記の[クイックスタート](#🚀-クイックスタート推奨)のコマンドを実行すると、自動的にリポジトリが作成される。

### 2.2 リポジトリをローカルにコピー

1. **GitHub Desktop でクローン**
   - 作成されたリポジトリの画面で `Code` ボタンをクリック
   - `Open with GitHub Desktop` をクリック
   - 自分のPCでリポジトリーを置くディレクトリを確認して `Clone` ボタンをクリック

### 2.3 VS Code で開く

1. **GitHub Desktop で VS Code を開く**
   - GitHub Desktop で `Open in Visual Studio Code` をクリック
   - 初回は devcontainer のセットアップが自動で実行される
   - LaTeX Workshop 拡張機能が自動的に利用可能

### 2.4 週報執筆環境の確認

自動セットアップにより以下が利用可能である。
- LaTeX コンパイル（ファイル保存で自動実行）
- textlint による日本語校正
- SyncTeX（.tex ファイルと PDF の相互ジャンプ）

### 2.5 毎週 tex ファイルを作成し編集する

1. **日付入りファイルの作成**
   - `20yy-mm-dd.tex` をコピーし、週間報告する日付入りのファイル名を作成
   - 例：2023/5/18 に報告するなら `2023-05-18.tex`

2. **VS Code で週報を編集**
   - VS Code で tex ファイルを編集
   - 必要に応じて画像ファイルなどを追加

3. **自動機能の活用**
   - ファイル保存で LaTeX が自動コンパイルされ PDF が生成される
   - textlint による自動文章チェック（結果は VS Code の「問題」に表示）
   - SyncTeX で .tex と PDF を相互ジャンプ可能

4. **プレビューで確認**
   - VS Code 上で PDF をプレビューし、内容を確認

### 2.6 GitHub への保存

1. **GitHub Desktop でコミット・プッシュ**
   - 週報完成後、GitHub Desktop でコミット・プッシュ
   - 自動で PDF も生成されリリースとして公開される

## 3. 週間報告の書き方 (20yy-mm-dd.tex の更新)

### 3.1 週間報告としての注意事項

- その週の研究・授業・就職活動などに関する出来事を記述すること
- 可能であれば図表も入れること

### 3.2 日本語的な注意事項

- である体で書くこと
- 長い文を書かず、適切に文を分割すること
  - 「が」で続けるのは良くない
- 適切な内容で段落を分割すること
- 文の途中に箇条書きを入れないこと
- すべての図や表は本文中で参照すること

### 3.3 LaTeX的な注意事項

- 図や表にはキャプションを入れる。入れ方は[example.tex](example.tex)を参考にすること
- 原則として、1文1行で記述すること
  - 差分が見やすいし、編集指示も出しやすい
- `\\` による改行は `tabular` 環境以外では原則として使用不可
